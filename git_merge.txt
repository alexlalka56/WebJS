Зараз мим вивчемо як вирішувати конфлікти між файлами які були створені або змінені в різних гілках.

Зараз виникла ситуація, що коли ми хочемо зробити merge з гілки task4 в гілку main, виникає конфлікт.
Для того щоб вирішити цей конфлікти ми можемо зробити merge локально за допомогою команди git merge task4, перед цим потрібно перейти в гілку main.
Вводимо команду git status щоб перевірити гілку, в якій ми знаходимось та чи є якізь зміни в ній.
За допомогою команди git checkout main(або назва іншої гілки) переходимо в наш бранч куди ми хочемо зробити merge.
Виконуємо команду git merge origin/task4 (або назва іншої гілки).

>>>>>>> origin/task4 - показує кінець коду гілки з якої робимо merge.

Видаляємо позначки точок, та комітимо наші зміни.
